# v1.3.4 (2026-06-16) — Windows hotfix

**Emergency fix for v1.3.3 regression on Windows + claudecode agent.** Single commit
delta from v1.3.3. Linux / macOS users and users running non-claudecode agent runtimes
are not affected by the v1.3.3 bug; this release is still safe to upgrade for them.

If you're on Windows running claudecode (any platform: Feishu / WeCom / WeChat /
Telegram / Discord / Slack / DingTalk / QQ / etc.), **upgrade now**.

## What's fixed

### Windows: messages silently never reply on v1.3.3 (#1376)

After upgrading to v1.3.3 on Windows, all incoming messages stopped producing a
reply. Symptoms reported by users:

- Bot sees the message but session never starts → no reply ever arrives
- Windows event log / stderr shows GBK-encoded `命令行太长。` (cmdline too long)
- `❌ 错误: session process is not running` in user-visible cards

**Root cause**: v1.3.3 expanded `core.AgentSystemPrompt()` from 2707 bytes to 9055
bytes (added cron/timer detailed instructions + an ASCII table). cc-connect spawns
`claude.exe` with `--append-system-prompt <inline 9KB>`, which busts Windows
`cmd.exe`'s 8192-byte total command-line limit → `claude.exe` refuses to start →
every session immediately dies. Linux / macOS exec argv has no such limit, so the
regression is Windows-only.

**Fix**: `claudecode` agent now passes the system prompt to `claude.exe` via a file
on disk (`--append-system-prompt-file <path>`) instead of inline on the command line.
- Default 99% path: cc-connect writes a single shared file at
  `<data_dir>/agent-prompts/cc-connect-system.md` (`~/.cc-connect/agent-prompts/cc-connect-system.md`
  by default) at startup; all sessions reuse it.
- Edge 1% path (Slack / Weixin / MAX / when user configures `append_system_prompt`):
  per-spawn temp file.
- **Prompt content is identical to v1.3.3** — only the delivery mechanism changed.
  Agent behavior does not regress.
- Cmdline now carries only the short file path (~80 bytes) → fits comfortably under
  Windows' 8192-byte limit no matter how large the prompt grows in the future.

## What's NOT in this release

Strictly hotfix scope. No new features, no other bug fixes, no behavior changes
outside of the cmdline delivery mechanism above.

If you were waiting on something else, it'll land in v1.3.5 or later.

## Upgrade notes

- **No config changes required.** Existing config files keep working unchanged.
- A new file `<data_dir>/agent-prompts/cc-connect-system.md` is generated on first
  startup. Safe to delete (cc-connect rewrites it next startup).
- If you set `append_system_prompt` in your claudecode config, behavior is unchanged
  (still merged into the prompt, just delivered via file).
- Downgrade is safe (config-compatible with v1.3.3 and earlier).

## Compatibility

- Windows / macOS / Linux: all green (Linux/macOS were unaffected; Windows is fixed).
- All agent runtimes: claudecode fixed; codex / opencode / qoder / cursor / gemini /
  kimi / pi / others unaffected (they don't use cmdline delivery).
- All platforms: unaffected (the bug was in agent process spawn, not in platform code).
